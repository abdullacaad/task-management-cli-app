GIT QUICK GUIDE FOR STUDENTS
============================

Prerequisites
-------------
1) Install Git from https://git-scm.com/downloads
2) Set your identity once:
   git config --global user.name "Your Name"
   git config --global user.email "you@example.com"

Create or Clone the Repo
------------------------
# Start a new repo in current folder
   git init

# Or clone an existing repo
   git clone <repo-url>
   cd <repo-folder>

Daily Workflow
--------------
1) Check status
   git status

2) See changes (diff)
   git diff                # unstaged changes
   git diff --cached       # staged changes

3) Stage files
   git add <file>
   git add .               # stage everything (use carefully)

4) Commit with a clear message
   git commit -m "Describe the change"

5) View history
   git log --oneline --graph --decorate

6) Push to remote (after adding remote)
   git remote add origin <repo-url>   # first time only
   git branch -M main                 # optional: rename master to main
   git push -u origin main            # first push sets upstream

7) Pull updates from remote
   git pull

Working Safely
--------------
- Use meaningful commit messages (present tense, short):
  Examples:
    git commit -m "Add task manager classes"
    git commit -m "Fix save error handling"
- Check status often: git status
- Avoid committing large/generated files; use .gitignore

Branching (optional but recommended)
------------------------------------
- Create and switch to a branch
    git checkout -b feature/add-tests
- Switch back to main
    git checkout main
- Merge your branch into main
    git checkout main
    git merge feature/add-tests

Fixing Mistakes (safe commands)
-------------------------------
- Unstage a file (keeps changes):
    git restore --staged <file>
- Discard local changes in a file (careful!):
    git restore <file>

Housekeeping
------------
- See configured remotes:
    git remote -v
- Rename current branch to main (if needed):
    git branch -M main

Typical Session Example
-----------------------
    git status
    git add task_manager.py
    git commit -m "Implement task deletion rollback"
    git push

Tips
----
- Commit small, logical changes frequently.
- Pull before you push to avoid conflicts.
- If a merge conflict happens, resolve files, then:
    git add <resolved-files>
    git commit
- To view commit details:
    git show <commit-hash>

Placeholders
------------
- Replace <repo-url> with your GitHub repository URL.
- Replace <file> with the actual filename you changed.
- Replace messages with a short, meaningful description.
